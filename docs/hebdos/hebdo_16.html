Filtrer le jeu de données

Définition du seuil de NDVI

En filtrant les pixels avec un NDVI > 0.7 avant le cyclone et en enlevant ceux situés sur la côte à une altitude < 50 m, on obtient de bien meilleurs résultats dans nos modèle. Cependant ce seuil de NDVI pré-cyclonique est plutôt arbitraire. Pour tenter de trouver un seuil acceptable cohérent, on fait un kmeans (même si la distribution n’est pas bimodale).

# K-means avec k=2
set.seed(42)
km <- kmeans(dta$dec_ndvi, centers = 2)
dta$cluster <- factor(km$cluster)

# Voir les centroïdes 
aggregate(dec_ndvi ~ cluster, data = dta, FUN = mean)

Le seuil de 0.7 n’était finalement pas si mal.

Prédictions sur jeu de test

On peut maintenant tester le modèle sur le jeu de test.

pred_test <- predict(m, newdata = dta_test)
dta_test$pred <- pred_test
residuals_test <- dta_test$delta_ndvi - pred_test
dta_test$residus <- dta_test$delta_ndvi - dta_test$pred


rmse_test <- sqrt(mean(residuals_test^2))
ss_res <- sum(residuals_test^2)
ss_tot <- sum((dta_test$delta_ndvi - mean(dta_test$delta_ndvi))^2)
r2_test <- 1 - ss_res / ss_tot

cat("RMSE test :", round(rmse_test, 4), "\n")

cat("R² test   :", round(r2_test, 4), "\n")

Extrapolation à tout Mayotte

Pour étendre notre modèle à tout Mayotte et essayer de prédire les dommages de Chido sur l’entièreté de l’île, il faut préparer les varibales explicatives : msw_max, altitude, tew, courbure. Il faut aussif aire attention à la normalisation des données qui avaient été faite dans le jeu de données de base.

# altitude
demMayotte <- rast("/home/abiton/Documents/dem_data/dem_mayotte.tif")
mask_land <- ifel(demMayotte > 0, 1, NA)
demMayotte <- demMayotte * mask_land

# msw
speed_layers <- pf[[grep("CHIDO_Speed", names(pf))]]
max_speed_raster <- max(speed_layers, na.rm = TRUE)

max_speed_raster <- project(max_speed_raster, demMayotte)
vmax_aligned <- crop(max_speed_raster, demMayotte)
mswMayotte <- mask(vmax_aligned, demMayotte)

# tew6 
tew6Mayotte <- computeTEW(sts, demMayotte, pf, product = "Summary", angle = 6, verbose = 0)

# courbure
penteM <- terrain(demMayotte, v= "slope")
curvMayotte <- terrain(penteM, v = "slope")

# stack 
stack_mayotte <- c(mswMayotte, tew6Mayotte, demMayotte, curvMayotte, penteM)
names(stack_mayotte) <- c("msw_max", "ang6", "altitude", "courbure", "pente")

# convertir chaque couche en numeric
stack_mayotte[["msw_max"]]  <- as.numeric(stack_mayotte[["msw_max"]])
stack_mayotte[["ang6"]]     <- as.numeric(stack_mayotte[["ang6"]])
stack_mayotte[["altitude"]] <- as.numeric(stack_mayotte[["altitude"]])
stack_mayotte[["courbure"]] <- as.numeric(stack_mayotte[["courbure"]])
stack_mayotte[["pente"]] <- as.numeric(stack_mayotte[["pente"]])
cols <- c("msw_max", "ang6", "altitude", "courbure", "pente")

params_scale <- lapply(cols, function(v) {
  list(mean = mean(dta[[v]], na.rm = TRUE),
       sd   = sd(dta[[v]],   na.rm = TRUE))
})
names(params_scale) <- cols

# normaliser
stack_scaled <- stack_mayotte
for (v in cols) {
  stack_scaled[[v]] <- (stack_mayotte[[v]] - params_scale[[v]]$mean) / params_scale[[v]]$sd
}
# prediction
pred_mayotte <- predict(stack_scaled, m, na.rm = TRUE)
ggplot() +
  geom_spatraster(data = pred_mayotte) +
  scale_fill_gradient2(
    low      = "darkred",
    mid      = "white",
    high     = "darkgreen",
    midpoint = 0,
    na.value = "transparent",
    name     = "Delta NDVI"
  ) +
  labs(title = "Prédit sur Mayotte (m)") +
  coord_sf() +
  theme_minimal()

Random Forest permet plus de nuances dans les prédictions du modèles, on peu notamment voir une perte de végétation moins importante dans le sud de l’île, ce qui est plus cohérent avec la trajectoire de Chido et ce qu’on peut voir visuellement.
